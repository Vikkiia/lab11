# Лабораторная работа 11

## Тема

Организация CI/CD и автоматизированного тестирования с использованием Jenkins.

## Выбранный стек

Node.js + npm. Проект представляет собой простую веб-страницу с формой заявки на настройку CI/CD. Для Jenkins подготовлен декларативный pipeline в файле `Jenkinsfile`.

## Состав проекта

- `src/index.html` - веб-страница с формой.
- `src/styles.css` - стили страницы.
- `src/formLogic.js` - логика нормализации и валидации формы.
- `src/app.js` - обработчик отправки формы в браузере.
- `tests/formLogic.test.js` - автотесты логики формы.
- `scripts/build.js` - сборка проекта в папку `dist`.
- `scripts/serve.js` - локальный запуск страницы.
- `Jenkinsfile` - описание CI/CD pipeline для Jenkins.

## Локальный запуск

```bash
npm install
npm test
npm run build
npm start
```

После запуска страница доступна по адресу `http://localhost:3000`.

## Настройка Jenkins

1. Установить Jenkins, Git и Node.js на машину агента Jenkins.
2. Разместить проект в Git-репозитории.
3. В Jenkins создать задачу `Pipeline`.
4. В разделе `Pipeline` выбрать `Pipeline script from SCM`.
5. Указать Git-репозиторий и путь к файлу `Jenkinsfile`.
6. Запустить сборку кнопкой `Build Now`.

## Этапы pipeline

1. `Checkout` - получение исходного кода из репозитория.
2. `Install` - установка зависимостей командой `npm ci`.
3. `Test` - запуск автотестов командой `npm test`.
4. `Build` - сборка статической веб-страницы командой `npm run build`.
5. `Post` - публикация содержимого папки `dist` как артефакта сборки.

## Ожидаемый результат

При успешном выполнении pipeline Jenkins должен показать зеленый статус сборки, а в артефактах задачи должна появиться собранная версия веб-страницы из папки `dist`.
